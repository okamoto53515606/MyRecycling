## 手順1: はじめに（個人・個人事業主向けの準備）

### 特定商取引法に基づく表記とは？

インターネットで商品やサービスを販売する場合、**特定商取引法（特商法）**に基づき、事業者の氏名・住所・電話番号などを明記することが法律で義務付けられています。

これは消費者保護のための制度ですが、個人で事業を行う場合、自宅住所や携帯電話番号を公開することに抵抗がある方も多いでしょう。

### Stripe審査で必要な情報

Stripeの本番環境を有効にするには、以下の情報が必要です：

| 項目 | 説明 |
|-----|------|
| 事業者名 | あなたの氏名または屋号 |
| 住所 | 事業所の住所（自宅または事業用住所） |
| 電話番号 | 連絡可能な電話番号 |
| 本人確認書類 | マイナンバーカードなど |
| 銀行口座 | 売上の振込先 |

### 個人情報を公開したくない場合の対策

自宅の住所や電話番号を公開したくない場合、以下のサービスを活用することで対応できます：

| サービス | 用途 | okamoの例 |
|---------|------|----------|
| IP電話サービス | 事業用電話番号の取得 | My 050 |
| バーチャルオフィス | 事業用住所の取得 | DMMバーチャルオフィス |

> **📌 ポイント**
> 
> これらのサービスを利用することで、特商法ページには自宅以外の住所・電話番号を記載できます。
> Stripe審査でも、事業用として登録した住所・電話番号を使用できます。

## 手順2: 事業用の電話番号を取得する

自宅とは別に、事業用の電話番号を取得します。ここでは「My 050」を例に説明します。

### My 050とは？

**My 050**は、ブラステル株式会社が提供するIP電話サービスです。

👉 https://www.brastel.com/my050/jpn/

| 項目 | 内容 |
|-----|------|
| 初期設定費 | 550円（返金不可） |
| 番号維持費 | 550円 / 6ヶ月毎（返金不可） |
| 月額基本料 | 無料（6ヶ月以上利用がない場合は解約） |
| 着信料 | 無料 |
| 発信料 | 固定電話への発信：約8円/3分程度 |
| 特徴 | 050番号が取得でき、スマホアプリで発着信可能 |

> **💡 なぜMy 050がおすすめ？**
> 
> - 月額固定費がかからない（6ヶ月毎に番号維持費550円は必要）
> - 050番号は「IP電話」として広く認知されている
> - 特商法の電話番号として使用可能
> - お客様からの問い合わせに対応できる

### 申込手順

1. My 050の公式サイトにアクセス

👉 https://www.brastel.com/my050/jpn/

2. アプリをダウンロードし、新規登録

3. **本人確認書類を提出**（以下のいずれか）
   - マイナンバーカード
   - 運転免許証
   - パスポート + 住民票（郵送）

> **⚠️ 注意**
> 
> 本人確認には数日〜1週間程度かかる場合があります。時間に余裕をもって申し込みましょう。

### okamoの場合の例

私（okamo）は以下の方法で本人確認を行いました：

- **提出書類**：マイナンバーカード + 住民票（郵送）
- **所要時間**：約5日間で本人確認完了
- **初期費用**：1,100円（初期設定費550円 + 番号維持費550円）
- **月額コスト**：0円（発信しない限り料金は発生しない。6ヶ月毎に番号維持費550円が必要）

電話番号取得後は、My 050アプリをインストールしておけば、いつでも着信を受けられます。

## 手順3: 事業用の住所を取得する

自宅とは別に、事業用の住所を取得します。ここでは「DMMバーチャルオフィス」を例に説明します。

### DMMバーチャルオフィスとは？

**DMMバーチャルオフィス**は、DMM.comが提供するバーチャルオフィスサービスです。

👉 https://virtualoffice.dmm.com/

| 項目 | 内容 |
|-----|------|
| 初期費用 | 5,500円（税込）程度 |
| 月額料金 | 660円〜（税込、プランにより異なる） |
| 利用可能住所 | 渋谷、銀座、大阪梅田など一等地 |
| 主なサービス | 住所利用、郵便物転送、法人登記（一部プラン） |

> **💡 なぜバーチャルオフィスがおすすめ？**
> 
> - 自宅住所を公開しなくて済む
> - 銀座や渋谷などの住所が使えて信頼感アップ
> - 郵便物は転送してもらえる
> - 特商法の住所として使用可能

### 申込手順

1. DMMバーチャルオフィスの公式サイトにアクセス

👉 https://virtualoffice.dmm.com/

2. 希望するプランと拠点を選択

3. 必要事項を入力して申し込み

4. **本人確認書類を提出**（2点必要）
   - マイナンバーカード
   - 運転免許証
   - パスポート
   - 住民票 など

> **⚠️ 注意**
> 
> 審査には数日〜1週間程度かかる場合があります。Stripeの本番申請前に余裕をもって準備しましょう。

### okamoの場合の例

私（okamo）は以下の条件で契約しました：

- **拠点**：渋谷
- **プラン**：ミニマムプラン（登記不可）年間契約
- **提出書類**：マイナンバーカード + 運転免許証
- **審査期間**：約3日で利用開始
- **初期費用**：13,420円（入会金5,500円 + 年間利用料7,920円）
- **月額換算**：約660円（税込）

住所は特商法ページやStripeの事業者情報として使用できます。

## 手順4: 特商法ページ等の内容を修正する

取得した電話番号と住所を使って、サイトの法的ページを更新します。

### 手順

1. あなたのサイトにアクセスし、管理者アカウントでログイン

2. 右上のアイコンをクリック → 「**管理画面**」をクリック

3. 左側メニューの「**サイト設定**」をクリック

4. 以下の項目を編集します

#### 特定商取引法に基づく表示

以下の情報を正確に記載してください：

| 項目 | 記載内容 |
|-----|----------|
| 販売業者 | あなたの氏名または屋号 |
| 所在地 | 事業用住所（バーチャルオフィスの住所） |
| 電話番号 | 事業用電話番号（050番号など） |
| メールアドレス | 連絡先メールアドレス |
| 販売価格 | 各商品ページに表示 |
| 支払方法 | クレジットカード |
| 支払時期 | 購入時に即時決済 |
| 商品引渡時期 | 購入完了後、即時閲覧可能 |
| 返品・交換 | デジタルコンテンツのため返品不可 |

#### プライバシーポリシー

- 個人情報の取扱いについて記載
- 必要に応じて、デフォルトの内容を修正

#### ご利用規約

- サービスの利用条件について記載
- 必要に応じて、デフォルトの内容を修正

5. 「**設定を保存**」ボタンをクリック

> **⚠️ 重要**
> 
> これらのページは**Stripe審査で確認される**ことがあります。
> 内容に不備があると審査が遅れたり、否認される可能性があります。
> **Stripe本番申請前に必ず完成させてください。**

## 手順5: Googleログイン画面のブランディングを登録

Googleログイン時に表示されるアプリ情報を設定します。これにより、ユーザーが安心してログインできるようになります。

### 手順

1. GCPコンソールにログイン

👉 https://console.cloud.google.com/

2. 左メニューから「**APIとサービス**」→「**認証情報**」をクリック

3. 画面上部の「**ブランディング**」タブをクリック（または「OAuth同意画面」から設定）

4. 以下の情報を登録します

![OAuth ブランディング設定](screenshot/google_oauth_branding.png)

| 項目 | 設定内容 |
|-----|----------|
| アプリ名 | あなたのサイト名（日本語可） |
| ユーザーサポートメール | 問い合わせ用メールアドレス |
| アプリのロゴ | サイトのロゴ画像（省略可） |
| アプリのホームページ | あなたのサイトURL |
| アプリケーションプライバシーポリシーのリンク | プライバシーポリシーページのURL |

5. 「**保存**」ボタンをクリック

6. 右上の「**ブランディングの確認**」ボタンをクリック

> **⚠️ 重要：「ブランディングの確認」を忘れずに！**
> 
> 「保存」ボタンを押した後、必ず右上の「**ブランディングの確認**」ボタンをクリックしてください。
> **このボタンを押さないと、Googleの審査が開始されません！**
> 忘れやすいポイントなので、必ず確認してください。

## 手順6: Stripeの本番化申請

Stripeダッシュボードから、本番環境の有効化（アカウント有効化）を申請します。

### 事前確認

申請前に、以下を必ず確認してください：

- [ ] 特商法ページが完成している
- [ ] プライバシーポリシーページが完成している
- [ ] 利用規約ページが完成している
- [ ] 事業用電話番号を取得済み
- [ ] 事業用住所を取得済み（バーチャルオフィスなど）
- [ ] 銀行口座情報を用意済み

> **⚠️ 重要**
> 
> 法的ページが不完全な状態で申請すると、審査が遅れたり否認される可能性があります。
> **必ず完璧な状態にしてから申請してください。**

### 参考リンク

Stripeアカウントの有効化について：
👉 https://docs.stripe.com/get-started/account/activate?locale=ja-JP

### 手順

1. Stripeダッシュボードにログイン

👉 https://dashboard.stripe.com/

2. 画面上部に表示される「**本番環境の有効化**」または「**今すぐ開始**」をクリック
   （表示がない場合は「設定」→「アカウントの詳細」から申請できます）

3. 以下の情報を入力します

#### ビジネス情報

| 項目 | 入力内容 |
|-----|----------|
| ビジネスの種類 | 個人事業主 または 個人 |
| 業種 | ソフトウェア、デジタルコンテンツ など |
| ビジネスのWebサイト | あなたのサイトURL |

#### 本人情報（本人確認書類と一致させる）

| 項目 | 入力内容 |
|-----|----------|
| 氏名 | あなたの氏名（本人確認書類と同じ） |
| 生年月日 | あなたの生年月日 |
| 自宅住所 | 本人確認書類に記載の住所 |

> **⚠️ 重要：本人確認書類との整合性**
> 
> 本人情報は、マイナンバーカードなどの**本人確認書類と同じ情報**を入力してください。
> Stripeは提出書類と入力情報を照合するため、一致しないと審査に通りません。

#### ビジネスの公開情報

| 項目 | 入力内容 |
|-----|----------|
| 事業所住所 | 事業用住所（DMMバーチャルオフィスなど） |
| 電話番号 | 事業用電話番号（My 050など） |

> **💡 ポイント：本人情報と公開情報の違い**
> 
> | 種類 | 用途 | 入力する情報 |
> |-----|------|-------------|
> | 本人情報 | Stripeの本人確認用（非公開） | 本人確認書類と同じ実際の情報 |
> | ビジネスの公開情報 | 顧客への表示・連絡先 | My 050やバーチャルオフィスの情報 |
> 
> 本人情報は審査のためにStripeが使用するだけで、顧客には公開されません。
> ビジネスの公開情報は、領収書や問い合わせ先として顧客に表示されます。

#### 本人確認書類

以下のいずれかをアップロードします：

- マイナンバーカード（okamoの場合はこれを使用）
- 運転免許証
- パスポート

#### 銀行口座情報

売上を受け取る銀行口座を登録します：

| 項目 | 入力内容 |
|-----|----------|
| 金融機関名 | 銀行名 |
| 支店名 | 支店名 |
| 口座種別 | 普通 または 当座 |
| 口座番号 | 口座番号 |
| 口座名義 | カタカナで入力 |

4. すべての情報を入力したら「**申請を送信**」ボタンをクリック

## 手順7: Stripe審査の結果を待つ

申請後、Stripeによる審査が行われます。

### 審査期間の目安

| 状況 | 期間 |
|-----|------|
| 標準的な場合 | 1〜3営業日 |
| 追加確認が必要な場合 | 1〜2週間 |

審査結果はメールで通知されます。

### 審査で確認されること

Stripeは以下のポイントを確認しています：

| 確認項目 | 詳細 |
|---------|------|
| ビジネスの実在性 | Webサイトが稼働しているか |
| 法的ページの整備 | 特商法、プライバシーポリシー等があるか |
| 本人確認 | 提出書類と入力情報が一致しているか |
| 事業内容の適正性 | 禁止されているビジネスに該当しないか |

### 審査に落ちた場合

万が一、審査に通らなかった場合は：

1. Stripeからのメールで**理由を確認**
2. 指摘された箇所を**修正**
3. **再申請**

> **💡 審査通過のコツ**
> 
> - 特商法ページに必要事項をすべて記載する
> - Webサイトが正常に動作していることを確認する
> - 提出書類と入力情報に相違がないようにする
> - 問い合わせ可能なメールアドレス・電話番号を登録する

### 審査中にできること

審査中も**サンドボックス環境**は引き続き利用できます。
本番の設定準備を進めながら、審査完了を待ちましょう。

## 手順8: Stripe本番環境の設定

審査が通ったら、本番環境の設定を行います。
これは[手順書Part2](setup2.md)の手順2〜7と同様の作業を**本番環境で**行います。

### 手順8-1: 本番環境のワークベンチを開く

1. Stripeダッシュボードにログイン

👉 https://dashboard.stripe.com/

2. 画面上部で「**サンドボックス**」ではなく「**本番環境**」が選択されていることを確認

> **⚠️ 注意**
> 
> サンドボックスと本番環境を間違えないように注意してください！
> 本番環境では実際のお金が動きます。

### 手順8-2: APIキーをメモする

1. ワークベンチの画面で、以下の2つのキーを確認します
   - **公開可能キー**（`pk_live_`で始まる）※サンドボックスは`pk_test_`
   - **シークレットキー**（`sk_live_`で始まる）※サンドボックスは`sk_test_`

2. それぞれの「コピー」ボタンをクリックして、**メモ帳に保存**しておきます

> **💡 注意：本番キーの識別方法**
> 
> | 環境 | 公開可能キーのprefix | シークレットキーのprefix |
> |-----|---------------------|------------------------|
> | サンドボックス | `pk_test_` | `sk_test_` |
> | 本番環境 | `pk_live_` | `sk_live_` |
> 
> 必ず`pk_live_`と`sk_live_`で始まるキーをコピーしてください！

### 手順8-3: Webhookを作成する

1. ワークベンチで「**Webhook**」タブを選択し、「**送信先を追加する**」ボタンをクリック

2. 以下を設定します
   - **APIバージョン**：最新バージョンを選択
   - 「**すべてのイベント**」タブで `checkout.session.completed` を検索
   - `checkout.session.completed` にチェックを入れる
   - 「**続行**」ボタンをクリック

3. 「**Webhookエンドポイント**」を選択し、「**続行**」ボタンをクリック

4. 「**エンドポイントURL**」に以下を入力し、「**送信先を作成する**」ボタンをクリック

```
https://あなたのドメイン/api/stripe/webhook
```

5. 作成完了後、「**署名シークレット**」を**メモ帳に保存**（`whsec_`で始まる）

### 手順8-4: 税率を作成する

1. 左メニューの「**商品カタログ**」から「**税率**」をクリック
2. 右上の「**税率を作成**」ボタンをクリック

3. 以下の通り入力し、「**税率を追加**」ボタンをクリック
   - **表示名**：消費税
   - **税率**：10（%）
   - **税率のタイプ**：内税

4. 作成された税率の「**コピー**」ボタンをクリックし、**税率ID**（`txr_`で始まる）を**メモ帳に保存**

### 手順8-5: 3Dセキュアの設定

1. 右上の**歯車アイコン**（設定）をクリック

2. 「**Billing**」→「**サブスクリプションとメール通知**」を開く

3. 「**Radar のルールに一致する Billing 支払いに 3D セキュアをリクエスト**」のチェックを**ON**にする

### 手順8-6: 領収書メール設定

1. 右上の**歯車アイコン**（設定）をクリック

2. 「**ビジネス**」→「**送信メール**」を開く

3. 以下を設定します
   - 「**決済成功時**」のチェックを**ON**
   - 「**領収書メールに登録番号を記載する**」を**ON**（登録番号がある方のみ）

4. 「**Billing**」→「**請求書**」を開く

5. 以下を設定して「**保存**」ボタンをクリック
   - 「**デフォルトの項目価格**」は「**税込み**」を選択
   - 「**納税者番号**」の欄で登録番号を登録（登録番号がある方のみ）

> **🎉 Stripe本番環境の設定完了！**  
> お疲れさまでした！Stripe側の設定はこれで完了です。

## 手順9: .envファイルを本番用に変更する

サンドボックス用のキーを本番環境用のキーに置き換えます。

### 手順

1. Firebase Studioを開く

👉 https://studio.firebase.google.com/

2. 右上の「**</>**」をクリック

3. 左側のエクスプローラーで「**.env**」ファイルをクリック

4. 以下の4つの項目を、**本番環境のキーに置き換え**ます

```
# stripe API 公開可能キー（クライアントサイドで使用）
NEXT_PUBLIC_STRIPE_PUBLIC_KEY="pk_live_xxxxxxxxx"

# stripe API シークレットキー（サーバーサイドのみ）
STRIPE_SECRET_KEY="sk_live_xxxxxxxxx"

# stripe 署名シークレット
STRIPE_WEBHOOK_SECRET="whsec_xxxxxxxxx"

# stripe > 商品カタログ > 税率 > 税率 ID を作成 (txr_からはじまるID)
STRIPE_TAX_RATES="txr_xxxxxxxxx"
```

| 環境変数名 | 置き換える値 |
|-----------|--------------|
| `NEXT_PUBLIC_STRIPE_PUBLIC_KEY` | 本番の公開可能キー（`pk_live_`...） |
| `STRIPE_SECRET_KEY` | 本番のシークレットキー（`sk_live_`...） |
| `STRIPE_WEBHOOK_SECRET` | 本番の署名シークレット（`whsec_`...） |
| `STRIPE_TAX_RATES` | 本番の税率ID（`txr_`...） |

> **⚠️ 重要：キーのprefixを確認！**
> 
> - サンドボックス：`pk_test_`、`sk_test_` で始まる
> - 本番環境：`pk_live_`、`sk_live_` で始まる
> 
> **必ず`_live_`のキーに置き換えてください！**

### 手順9-2: 本番サイトに反映する

1. 右上の「**Publish**」ボタンをクリック

2. 「**Publish**」ボタンをクリック

> **⏰ デプロイには数分かかります**  
> 完了するまでお待ちください。

## 手順10: 動作確認

本番環境で決済機能が正しく動作するかテストします。

> **⚠️ 重要：本番環境では実際の課金が発生します！**
> 
> サンドボックスとは異なり、**本番環境では実際のお金が動きます**。
> テストには**自分の本物のクレジットカード**を使用し、**実際の課金が発生する**ことを理解した上でテストしてください。

### 事前確認

> **⚠️ 管理者アカウントでテストしないでください**
> 
> 管理者アカウントでログインしている場合、**課金なしで有料記事が閲覧できてしまいます**。
> **管理者とは別のGoogleアカウント**でテストしてください。

### 手順

1. ブラウザの**シークレットウィンドウ**を開く

> **💡 シークレットウィンドウの開き方**
> - **Chrome**: `Ctrl + Shift + N`（Mac: `Cmd + Shift + N`）
> - **Firefox**: `Ctrl + Shift + P`（Mac: `Cmd + Shift + P`）
> - **Edge**: `Ctrl + Shift + N`

2. あなたのサイトにアクセスし、**管理者とは別のGoogleアカウント**でログイン

3. 有料記事をクリック

4. 「**購入する**」ボタンをクリック

5. **自分の本物のクレジットカード情報**を入力し、「**支払う**」ボタンをクリック

6. 支払い完了画面で「**領収書を表示・ダウンロード**」をクリック

7. **領収書が表示されれば成功です！** 🎉

8. 登録したメールアドレスに**領収書メールが届く**ことを確認

> **📌 本番環境での動作確認ポイント**
> 
> | 確認項目 | 期待される結果 |
> |---------|---------------|
> | 決済画面が表示される | Stripe Checkoutページが開く |
> | 決済が完了する | 支払い完了画面が表示される |
> | 記事が閲覧できる | 有料記事の全文が表示される |
> | 領収書メールが届く | 購入者のメールアドレスに自動送信される |

## ❓ トラブルシューティング

### よくある問題と対処法

| 症状 | 原因 | 対処法 |
|-----|------|-------|
| 決済画面が表示されない | 公開可能キーの設定ミス | `.env`の`NEXT_PUBLIC_STRIPE_PUBLIC_KEY`が`pk_live_`で始まるか確認 |
| 決済後に記事が読めない | Webhook設定のミス | 本番用のWebhookが正しく設定されているか確認 |
| 「無効なAPIキー」エラー | サンドボックスと本番のキー混在 | すべてのキーが本番用（`_live_`）になっているか確認 |
| 領収書メールが届かない | 領収書メール設定がOFF | Stripe設定の「決済成功時」がONになっているか確認 |
| IP制限で管理画面に入れない | 自分のIPアドレス設定ミス | Firebase Studioから`.env`を直接編集 |

### チェックリスト

問題が発生した場合は、以下を順番に確認してください：

- [ ] `.env`ファイルのキーがすべて本番用（`_live_`）になっているか
- [ ] WebhookのエンドポイントURLが正しいドメインになっているか
- [ ] 税率IDは本番環境で新規作成したものか
- [ ] Publishボタンでデプロイを実行したか
- [ ] Stripe本番環境の審査が完了しているか
