# 個人開発企画書：okamoのリサイクル（仮） v1.3

**〜 家庭の不用品をご近所受取で販売するシステム 〜**
**場所：国立周辺エリア限定（●●●公園 または 国立駅前）**

## 1. 開発の動機・コンセプト
* **「もったいない」の解消：**
* まだ使える子供服、ゲームソフト、ガジェットなどを、ゴミにするのではなく、近所の誰かに有効活用してもらいたい。
* **利益より「循環」と「手間削減」：**
* 高値で売りたいわけではない。安くて良いので、メルカリのような「梱包・発送・送料計算」の手間をゼロにしたい。
* **エンジニアリングによる解決：**
* 既存の対面取引（ジモティー等）の課題である「現金のやり取り」「ドタキャン」「対面の不安」を、自作システムと決済技術（Stripe）で解決する。

## 2. 私の「こだわり」（既存プラットフォームとの差別化）
### ① 梱包・配送ゼロ（Eco & Time-saving）
* ダンボールもプチプチも不要。そのまま渡す。
* ゴミが出ず、梱包作業の時間も取られない。

### ② 完全キャッシュレス＆安心決済（Safe & Smart）
* **「受け渡し完了」まで決済は確定しない：**
* 予約時はあくまで「枠の確保（オーソリ）」のみ。
* 現物を見て問題なければ、その場で管理画面から決済を確定させる。
* 対面での現金のやり取りは一切なし。お釣りや値切りのストレスを排除。

### ③ プラットフォーム中抜きなし（Direct）
* 手数料10%を取られず、その分安く提供できる。
* 自分のシステム（Stripe + Firebase）を使うため、売上サイクルも自分でコントロール可能。

### ④ 安心の「14日間返品保証」（Trust）
* 「安かろう悪かろう」にはしたくない。
* 受け渡し後、自宅でじっくり動作確認してもらい、万が一気に入らなければ返品・返金に応じる（システムで対応）。

## 3. 取引フロー（UX設計）

1. **出品・閲覧**
* okamoがサイトに不用品を登録。
* 購入希望者がサイトで商品を閲覧。
2. **予約・決済（オーソリ）**
* **日時選択：** **「次の土日（週末）」のみ**選択可能。
* *理由：Stripeのオーソリ有効期限（7日間）切れを防ぐため。*
* **場所選択：** 「北第一公園」または「国立駅前（電車利用を想定）」。
* **仮売上：** クレジットカードで**「仮売上（オーソリ）」**を実行。枠だけ確保する。
3. **承認・確定**
* okamoに通知が飛び、日時を承認。購入者に場所と時間の確定メール送信。
* 購入者の**「マイページ」**に予約情報が反映される。
4. **雨天・都合が悪い場合（キャンセル・再予約）**
* 悪天候や急用時は、購入者が**マイページからキャンセル**を実行。
* システムが即座に「オーソリ解放（キャンセル）」を実行（**手数料はかかりません**）。
* 商品ステータスが「予約済み」から「販売中」に戻るため、必要であれば次週以降の日程で再予約してもらう。
5. **受け渡し（対面）**
* 指定場所で合流。品物の状態を確認してもらう。
* 問題なければ、okamoが管理画面で**「受渡完了」**ボタンを押す。
* システムがStripeの**「売上確定（Capture）」**を実行。ここで初めて課金される。
6. **アフターフロー（14日間）**
* **問題なしの場合：** 何もしなくてOK。
* **返品希望の場合：** マイページから**「返品依頼」**ボタンを押下 → 再度日時を調整して公園等で受け渡し（返品） → okamoが管理画面から**「返金（Refund）」**実行。

## 4. 集客・マーケティング戦略（ドッグフーディング）

本プロジェクト自体を、運用中の個人メディア「homepage」の**実験コンテンツ（社会実験・OSS化）**として位置づけ、集客と信頼獲得を狙う。

* **コンテンツ化：**
* 「エンジニアが自宅の不用品を売るために、ご近所メルカリを自作してみた」等の記事を執筆。
* 開発プロセス、コード、運用ノウハウ（特商法やStripe審査など）を**OSS/ナレッジとして全公開**する。
* **信頼性の担保：**
* 「どこの誰かわからない人」ではなく、「技術情報を発信しているエンジニアの実験」として認知してもらうことで、近隣住民（特にリテラシーの高い層）の心理的ハードルを下げる。
* **告知：**
* メディアサイト内での告知を通じて流入を図る。

## 5. システム要件（MVP: Minimum Viable Product）

### 必須機能
#### ユーザー（購入者）側
* 商品一覧・詳細表示
* 予約フォーム（Stripe組み込み）
* **日時選択（Datepicker）：直近の土日（9:00-17:00）のみアクティブにするロジック**
* 特定商取引法に基づく表記ページ
* **マイページ機能**
* **予約確認：** 現在の予約状況（日時・場所）の表示。
* **キャンセル実行：** 雨天や都合が悪くなった際に利用。Stripeオーソリを解放し、商品のステータスを「予約可能」に戻す。
* **返品リクエスト：** 受取後の返品予約機能（予約フローと同様に日時を調整）。

#### 管理者（okamo）側
* 商品登録・編集・削除
* 予約一覧・承認機能
* **管理キャンセルボタン**（ユーザーからの連絡等で代理キャンセルする場合用）
* **受渡実行ボタン**（Stripe Capture API：売上確定）
* **返金実行ボタン**（Stripe Refund API）

## 6. 法的・規約上の注意点

### 特定商取引法
* **表記義務：** 既存メディアサイト運営のために取得済みの**「バーチャル住所・電話番号」**を流用する。
* **個人情報：** 自宅住所を公開せず、法的な要件を満たす。
* **返品・交換について：** 「商品引き渡しから14日以内であれば、理由を問わず返品を受け付けます。ただし、返品時の受け渡し場所は指定場所（国立周辺）に限ります」

### 運用ルール（キャンセルポリシー）
* **雨天対応：** 無理に決行せず、柔軟にリスケジュール（再予約）を行う旨を明記。
* **キャンセル料：** 受け渡し前のキャンセル（オーソリ解放）は無料とする。

### 所得区分（生活用動産）
「個人の不用品（生活用動産）の譲渡」なので、消費税法上は「不課税取引（消費税の対象外）」となる。帳簿上は明確に「事業主借 / 譲渡所得」として分けて管理。

### Stripeでの設定方法
Stripeのダッシュボードで、今回の不用品販売専用の**「税率（Tax Rate）」**を作成し、Checkout Sessionを作成する際に、この「0%の税率オブジェクト」を紐付け。
*   **種類:** `custom`（カスタム）
*   **名称:** `不課税` または `非課税`（英語名: `Non-taxable`）
*   **税率:** **0%**
*   **計算タイプ:** 外税/内税どちらでもOK（0%なので結果は同じ）
