## 調査結果

**原因：`.product-detail__content-body` にテーブル用のCSSが一切定義されていない**

### マークダウンの表示経路

- product-display.tsx で `<ReactMarkdown remarkPlugins={[remarkGfm]}>` を使ってマークダウンを描画
- `remarkGfm` プラグインにより GFM テーブル構文（`| col1 | col2 |`）は `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>`, `<td>` タグに正しく変換される
- 変換された HTML は `.product-detail__content-body` の中に出力される

### CSSの状態

globals.css:
```css
.product-detail__content-body {
  line-height: 1.8;
  font-size: 0.95rem;
  /* 'dangerouslySetInnerHTML'で出力されるHTMLタグのスタイル */
}
```

- `.product-detail__content-body` にはテーブル関連のスタイルが**一切ない**
- グローバルCSSのリセット（`* { margin: 0; padding: 0; }`）により、`<table>` のデフォルトスタイルも打ち消されている
- 結果として、テーブルに**罫線なし・パディングなし・背景色なし**の状態で表示される

### 既存のテーブルスタイル

CSS内には以下の2箇所にテーブルスタイルがありますが、どちらも別用途です：

1. **`.mypage__table`** ([globals.css L1605](src/app/globals.css#L1605)) — マイページの注文一覧用
2. **`.markdown-content table`** — **存在しない**（`.markdown-content` はモーダル内のマークダウン用だが、テーブルスタイルは未定義）

### 不足しているCSS

`.product-detail__content-body` に以下のようなテーブルスタイルを追加する必要があります：

- `table` — `width: 100%`, `border-collapse: collapse`, `margin` の設定
- `th`, `td` — `border`, `padding` の設定
- `th` — 背景色、フォントウェイトの設定
- レスポンシブ対応（横スクロール用の `overflow-x: auto` ラッパーなど）

**結論：ご推測の通り、globals.css にテーブル用のCSSが不足していることが原因です。**修正指示をいただければ対応します。
